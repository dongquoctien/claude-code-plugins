# Feature Mockup — Status

Show the current state of a feature: what's been done, what's pending, what's the suggested next step. Reads the feature's `.timeline.json` (auto-maintained by other skills) and prints a human-readable summary.

## Step 1 — Load config

Read `.claude/feature-mockup.json`. If missing, stop with:
> "Run `/feature-mockup:init` first."

Extract `outputDir` and `workingLanguage`.

## Step 2 — Resolve feature

Two modes:

### 2a. Feature name provided in `$ARGUMENTS`

Resolve `featureDir = {outputDir}/{$ARGUMENTS}`.

If `featureDir` does not exist:
> "No mockup found at `{featureDir}`."

### 2b. No argument — list all features

Glob `{outputDir}/*/.timeline.json`. For each match, read the timeline and print a one-line summary:

```
Features in this project (5):

  booking-cancel-flow     fixed ✓ · last activity 2 hr ago · 0 pending
  hotel-content-mgmt      verified · last activity 1 day ago · 3 P0 + 2 P1 pending
  bulk-edit-prices        drafted · last activity 3 days ago · no verify yet
  user-roles              deployed · last activity 1 week ago · 0 pending
  invoice-list            drafted · last activity 2 weeks ago · no verify yet

Run `/feature-mockup:status <feature-name>` to see details.
```

If no features have a timeline (maybe none yet, or pre-v0.21 prototypes without timeline), print:
> "No tracked features yet. Run `/feature-mockup:make <feature>` to start."

## Step 3 — Show feature detail (when argument provided)

If `{featureDir}/.timeline.json` does not exist:
> "Feature `{feature}` exists but has no timeline yet. The prototype was created before v0.21 OR by manual edits. Run `/feature-mockup:make {feature}` again to generate one, or proceed with other commands which will start logging from now."

Otherwise, simply print the contents of `{featureDir}/STATUS.md` directly. STATUS.md is auto-regenerated by every skill, so it's always current.

If STATUS.md is missing for some reason but `.timeline.json` exists, regenerate it:

```bash
node {pluginRoot}/scripts/timeline.mjs regenerate-status --feature-dir "{featureDir}"
```

Then print STATUS.md.

## Step 4 — Suggest next action (when relevant)

After printing STATUS.md, the file already includes a "Suggested next step" section. Skip duplicating it.

When `currentPhase === 'fixed'` and there's a pending P0 from a prior verify (rare race), surface a warning:
> "STATUS shows phase 'fixed' but {N} P0 issues still pending — consider re-running verify to confirm."

## Step 5 — Optional: trim old events

When `events.length > 100`, ask:
> "Timeline has {N} events. Archive entries older than 30 days to `.timeline-archive/`? (yes / skip)"

If yes:

```bash
# Move events older than 30 days into a dated archive file, keep newer in main timeline
node -e "
const fs = require('fs');
const t = JSON.parse(fs.readFileSync('{featureDir}/.timeline.json', 'utf8'));
const cutoff = new Date(Date.now() - 30*24*3600*1000).toISOString();
const old = t.events.filter(e => e.ts < cutoff);
const recent = t.events.filter(e => e.ts >= cutoff);
if (old.length === 0) { console.log('nothing to archive'); process.exit(0); }
const archDir = '{featureDir}/.timeline-archive';
fs.mkdirSync(archDir, { recursive: true });
const archFile = archDir + '/' + new Date().toISOString().slice(0,10) + '.json';
fs.writeFileSync(archFile, JSON.stringify({ archivedAt: new Date().toISOString(), events: old }, null, 2));
t.events = recent;
fs.writeFileSync('{featureDir}/.timeline.json', JSON.stringify(t, null, 2));
console.log('archived ' + old.length + ' events to ' + archFile);
"
```

## Edge cases

- **Multiple features with same name** — not possible; folder names are unique.
- **Stale STATUS.md after manual edits** — user can run `/feature-mockup:status <feature>` which auto-regenerates if missing. To force regenerate even when present, accept a `--refresh` flag (future).
- **Corrupt timeline.json** — if JSON parse fails, print the file path + ask user to fix or reset.
- **Events from skills that pre-date v0.21** — timeline starts empty. Show note: "Timeline starts here — earlier work is not tracked."

## Done

End the skill. The user has a clear picture of where they left off and what to do next.
